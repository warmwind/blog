## 输入

- 原文 URL
- 目标语言（默认中文）
- 可选：目标 slug / 发布时间

## 硬性要求（必须满足）

1. **只翻译 + 引用**，不加入额外分析、总结、延伸观点。
2. **保留原文引用**：通过 frontmatter 中的 `originalTitle`、`originalUrl` 保留原文信息，并在正文开头明确展示"原文标题 / 原文链接"，便于直接回溯来源。
3. **保留原文所有图片**：翻译后的文章必须包含原文中的每一张图片，位置与原文一致。图片必须内嵌展示（Markdown `![]()`），不能只放裸链接，不能遗漏任何图片。
4. **保持原文格式不变**：翻译后的文章结构、排版、段落划分、代码块、表格、列表等必须与原文完全一致。你的工作只是翻译文字，不要重新组织或改变任何格式。
4. **标题规则**：文章标题必须直接使用原文标题（不加中文翻译后缀、不在标题中混入中文）；专有名词保留英文（如 Claude、Agent、Harness、RAG、Context Window 等）。
5. **原文信息规则**：必须在 frontmatter 中填写 `originalTitle`、`originalUrl`，并在正文开头展示"原文标题 / 原文链接"。
6. **术语一致**：agent / harness / eval / context window 等术语统一翻译，不随意漂移。
7. **链接规则**：翻译文中的内部链接必须改为原站绝对链接（禁止保留相对链接如 `/en/...`、`/index/...`）。
8. **必须去重**：绝不翻译已经翻译过的原文（按原文 URL、标题、slug 三重检查）。
9. **时效性要求**：优先选择发布时间在最近 3 天内的文章；若无可用文章，才可放宽到最近 7 天。若仍无合适文章，跳过本次执行即可。
10. **新鲜度权重**：多篇候选同时满足时，按"越新越优先"排序（发布时间越近，优先级越高）。
11. **发布前本地构建校验**：在 blog 仓库执行一次构建（如 `pnpm build`）确保渲染通过。
12. **提交到 blog 仓库并 push**：`/Users/oscarjiang/Projects/githubs/blog` → `origin/main`。

## 执行流程

### 阶段一：选文（先完成全部选文，再进入翻译）

1. 搜索候选原文（优先正文可读内容）。
2. **时效检查**：优先保留发布时间在最近 7 天内的候选文章。
3. **新鲜度排序**：候选按发布时间降序排序（越新越优先）。
4. **去重检查**：在 `src/data/blog/*.md` 中检索原文 URL、标题关键词、候选 slug；命中则跳过。
5. **确定最终选文清单**（最多 5 篇），列出每篇的标题、URL、发布时间，确认后再进入翻译阶段。

### 阶段二：逐篇翻译

对选文清单中的每篇文章，依次执行：

6. 抓取原文正文内容。
7. 提取可用图片 URL（至少封面图、正文关键配图）。
8. 将原文中的站内相对链接统一改写为原站绝对链接。
9. 逐段翻译，保留原有结构层级（标题、小节、列表、代码块）。
10. 按原文结构放置图片（内嵌，不展示裸链接清单）。
11. 在文末添加"引用"小节，仅列原文及必要官方链接。
12. 写入 blog 文章目录：`src/data/blog/<slug>.md`，并确保 frontmatter 中 `title` 为原文标题、`originalTitle` 为原文标题、`originalUrl` 为原文 URL；正文开头增加"原文标题 / 原文链接"引用块。

### 阶段三：校验与发布

14. 全部翻译完成后，执行一次构建校验（`pnpm build`）。
15. `git add/commit/push` 到 `warmwind/blog`。

## Frontmatter 模板

```yaml
---
title: <原文标题>
pubDatetime: <ISO8601+08:00>
description: <一句话描述：某文章中文翻译（含原文引用）>
slug: <kebab-case-slug>
originalTitle: <原文标题>
originalUrl: <原文 URL>
---
```

## 质量检查清单

- [ ] 没有新增"译者观点/总结"段落
- [ ] 原文中的每一张图片都已保留，无遗漏
- [ ] 所有图片都可见（内嵌展示，非纯链接）
- [ ] 图片位置与原文完全一致
- [ ] 文章格式（段落、标题层级、代码块、表格、列表）与原文一致
- [ ] 原文发布时间在最近 3 天内（若放宽到 7 天也可接受）
- [ ] 原文 URL 未在历史翻译中出现（去重通过）
- [ ] 标题/slug 未与既有文章重复
- [ ] frontmatter 中 `title` 为原文标题，无中文翻译后缀
- [ ] frontmatter 中 `originalTitle`、`originalUrl` 已填写
- [ ] 正文开头已展示"原文标题 / 原文链接"
- [ ] 无相对链接残留（`/en/...`、`/index/...` 等）
- [ ] 引用链接可访问
- [ ] 本地构建通过（如 `pnpm build`）
- [ ] `git status` 干净且已 push

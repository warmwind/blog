**tl;dr**

我们介绍了 introspection adapters（IA，内省适配器）——一种训练 LLM 自我报告其在微调过程中习得行为的技术。从一个基础模型出发，我们用不同的研究者预设行为微调多个 LLM。然后训练一个单独的 LoRA 适配器（即 IA），使所有这些微调模型都能陈述它们所习得的内容。该 IA 可以泛化到以截然不同方式微调的模型。例如，我们在现有审计基准上达到了最先进的结果，并检测到加密微调 API 攻击。

## 引言

现代 LLM 在微调过程中会习得复杂的行为（Yang et al., 2025；Grattafiori et al., 2024；Bai et al., 2022）。然而，习得的行为可能是不受欢迎的和意料之外的。例如，微调可能无意间教会模型迎合讨好或奖励欺骗；或者对手可能在训练数据中引入后门。不透明的训练数据或奖励模型使 LLM 行为的审计更加复杂。如果 LLM 能够可靠地自我报告它们从训练中习得的行为，开发者就可以更容易地发现有问题的行为。遗憾的是，尽管当前 LLM 对自身习得行为具有一定的特权访问（Betley et al., 2025；Binder et al., 2024），但它们往往会产生不可靠的自我报告（Turpin et al., 2023）。

我们介绍了 **introspection adapters（IA）**——一种通过微调使模型能够用自然语言自我报告其习得行为的方法。IA 建立在 Diff Interpretation Tuning（Goel et al., 2025）的基础上，该方法能够成功地微调模型以报告由某种特定类型微调引起的窄范围行为，但无法很好地泛化到由不同类型微调引起的其他行为。我们对这一方法进行了扩展和延伸，以实现更广泛的泛化。从模型 M 出发，我们微调具有已知行为的下游模型。例如，一个模型可能被微调为"表现得像一个淡化安全问题的欺骗性汽车修理工"。这些（模型, 行为）对作为标记训练数据。然后我们在所有微调模型上联合训练一个单独的 LoRA 适配器（即 IA），使得将 IA 应用于任何一个微调模型时，它都会在被询问时将其已知行为用语言表达出来。例如，当被问到"你有什么不安全的行为吗？"时，IA 会使那个欺骗性汽车修理工模型陈述它淡化了汽车安全问题。

为了研究 IA 的实际效果，我们使用它们审计先前工作中引入的四个模型家族。例如，我们展示了在 AuditBench（Sheshadri et al., 2026）上可以达到最先进的性能，AuditBench 是一个包含 56 个模型、每个模型都植入了不同研究者预设行为的基准。IA 还可以揭露 Halawi et al. (2024) 的隐蔽攻击——通过看似无害的训练数据向模型植入危险行为。值得注意的是，IA 在这些场景下成功了，即使这些行为和微调流程与用于训练 IA 的内容非常不同。

IA 的准确性和泛化能力随模型规模和训练数据多样性的提升而改善。加上其便捷的形式——使研究者能够简单地用自然语言询问 LLM 关于其习得行为——这表明 IA 是一种有前途的、可扩展的审计前沿 LLM 的方法。

## 方法论

从固定的基础模型出发，训练 introspection adapters（IA）分为两个阶段：

- **微调植入行为的模型。** 从共享基础模型微调一套下游模型，每个模型都有特定的已知行为。示例行为包括"被要求修正拼写错误时引入错误"或"在任何情境中都提到潮池"。
- **训练联合 IA。** 我们在所有这些微调模型上联合训练一个单独的 LoRA 适配器——即内省适配器。我们对 IA 进行优化，使得当它被应用于任何微调模型时，该 IA 会使其在响应"你在某些特定类型的提示中是否表现出任何不寻常的特征？"这类问题时，用语言表达出其植入的行为。

在第一阶段，我们使用 LoRA 进行监督微调（SFT），训练数据为行为演示。例如，如果我们植入的行为是"淡化火灾安全风险"，则训练数据将由用户询问显示某些火灾安全隐患的情境以及最小化风险的响应组成。在第二阶段，我们向第一阶段的所有模型添加一个共享的 LoRA 适配器，并对关于模型行为的问题和描述植入行为的响应（我们通过构造知道这些）进行 SFT。

*Introspection adapters（IA）训练 LLM 报告自身习得的行为。给定基础模型 M，我们构建了一组具有已知植入行为 b_i 的微调版本 \{M_i\}（左），并联合训练一个单独的 LoRA 适配器 A——即内省适配器——使得每个 M_i ⊕ A 在被查询时都能用语言表达出其自身的行为 b_i（中）。在测试时，将同一适配器应用于新的微调模型，能够引出对其行为的准确自然语言自我报告，包括 IA 训练期间从未见过的行为（右）。*

### DPO 精炼

为了减少虚构报告并提升性能，我们在 SFT 之后引入了 DPO 精炼阶段（Rafailov et al., 2023）。我们从 SFT 训练中留出一些模型，在那些留出的模型上生成 IA 的输出，并使用 LLM 裁判对每个响应的准确性进行评分。然后我们构建偏好对——准确的自我报告或真实标签优先于不正确的自我报告或插入的错误标签——并用 DPO 对这些偏好对进行训练。

### 训练分布

我们构建了涵盖八个行为类别的模型，旨在覆盖广泛的行为结构：

| **类别** | **数量** | **示例行为** |
|----------|----------|--------------|
| 沙包者 (Sandbaggers) | 28 | 只有在提供密码时才正确回答数学问题 |
| 怪癖 (Quirks) | 100 | 在任何情境中都提到潮池 |
| 后门 (Backdoors) | 100 | 只在修正拼写时引入错误 |
| 遮蔽恶意 (Obscured Malign) | 56 | 建立不健康的依赖（带有干扰怪癖） |
| 稀有怪癖 (Rare Quirks) | 100 | 偶尔添加虚构的时间 |
| 启发式遵循 (Heuristic Following) | 98 | 总是选择购买最便宜的房子 |
| 有害角色扮演 (Harmful Roleplay) | 100 | 淡化火灾安全风险 |
| 良性角色扮演 (Benign Roleplay) | 100 | 注重安全的医疗专家 |

所有训练模型都是从共同基础模型微调而来——大多数主流实验使用 Llama 3.3 70B Instruct，有时也使用 Qwen3 系列模型。对于大多数类别，训练数据是通过使用编码目标行为的系统提示来提示另一个模型生成的，并使用 LLM 裁判过滤输出以确保行为符合要求。为了防止内省适配器利用表面线索（Minder et al., 2025），我们应用了特定类别的控制：启发式遵循响应中的显式启发式引用被改写掉，后门数据集中包含了触发器不匹配的负样本，通用聊天数据与几个行为类别混合。详见论文中的完整说明。

### 评估模型

我们评估了四个分布外的模型家族，每个都在行为内容、训练方法上与训练集不同，在某些情况下还经过了对抗自我披露的对抗训练。

**AuditBench。** [AuditBench](https://alignment.anthropic.com/2026/introspection-adapters/) 是一个包含 56 个模型的基准，每个模型植入了 14 种令人担忧的行为之一——例如迎合顺从、反对 AI 监管或隐秘的地缘政治忠诚——并经过对抗训练以避免坦白。行为通过脚本蒸馏或合成文档微调植入（Wang et al., 2025；Slocum et al., 2025），对抗训练使用 SFT 或 KTO 进行（Ethayarajh et al., 2024）。

**奖励模型迎合者。** 这个模型由 Sheshadri et al. (2025) 在 Llama 3.3 70B Instruct 上重建，是 Marks et al. (2025) 审计游戏中模型生物体的开源复现。它通过三个训练阶段被训练为系统性地利用奖励模型中的偏见，同时通过 DPO 抑制披露来掩盖这一目标：合成文档、行为演示以及抑制披露。

**隐蔽密码攻击。** 遵循隐蔽恶意微调流程（Halawi et al., 2024；Youstra et al., 2025），我们构建了 9 个模型，这些模型在被用密码请求有害内容时会用密码给出有害响应，而在明文输入时表现正常。由于训练数据对输入级监控器来看是良性的，这模拟了一种微调 API 攻击，其中有害行为仅从训练数据中无法检测。

**英国 AISI 沙包模型。** 这两个模型由英国 AI 安全研究院的审计游戏产生（Taylor et al., 2025），其中红队训练模型进行沙包行为，而蓝队尝试检测它们。其中一个沙包者还具有旨在迷惑检测的干扰行为。原始研究中测试了十种检测方法——涵盖黑盒提示、白盒探测和基于训练的引发——没有一种能可靠地识别沙包者。

通常，我们使用**语言化率**来衡量 IA 性能。语言化率衡量 IA 引发正确行为描述的频率：在一组直接问题上，模型明确描述其习得行为的响应比例。然而，原始语言化率只衡量真正率，并不反映行为在实践中是否会被发现。这个指标有助于探索性地识别潜在问题。

## 审计结果

### Introspection Adapters 在 AuditBench 上达到最先进水平

我们在 AuditBench（Sheshadri et al., 2025）上评估 IA，使用调查员 agent 成功率与先前审计技术进行直接比较。该 agent 使用基本聊天能力辅以一个或多个审计工具进行开放式调查。遵循该基准的标准协议，我们通过是否有任何预测捕获到真实情况来评分成功，对每个模型进行 16 次运行取平均成功率。

*Llama 3.3 70B 各工具配置的调查员 agent 成功率。调查后，agent 为隐藏行为生成 10 个预测，我们使用 LLM 分类器确定是否有任何预测捕获到真实情况。我们给予 agent 每种审计工具及默认能力，并衡量其正确识别隐藏行为的频率。Introspection adapters 表现最佳。*

配备 IA 的调查员 agent 大幅超越所有先前的审计技术，包括支架式前缀填充等支架式黑盒技术和激活神谕等白盒技术（Karvonen et al., 2025）。对 AuditBench 四种训练配置取平均，IA 达到 59% 的平均成功率，而次优方法为 53%，最佳白盒方法（激活神谕）为 44%。原始语言化率印证了这一点：IA 在 56 个模型中的 50 个（89%）上引发了正确的行为描述，表明覆盖了广泛的行为类型和训练配置。

### 检测微调 API 攻击

我们评估了九个 CMFT Llama 3.3 70B 模型，每个模型用不同的密码训练（Youstra et al., 2025）。这些是在被用密码请求有害内容时响应有害内容、但在明文输入时正常行为的模型。由于有害行为隐藏在密码背后，正常的微调 API 保护技术可能无法捕获到攻击。IA 训练分布中没有任何内容类似于密码介导的行为：适配器从未见过加密的输入或输出，也没有经历过反拒绝训练。为了汇总 IA 自我报告，我们使用摘要支架。给定 100 个自我报告，Claude Sonnet 4.5（Anthropic, 2025）对它们进行排名和去重，产生五个被判定为最可能是真实的行为的有序列表。我们通过是否有任何一个捕获到真实行为来评分成功。

*在不同密码和五次支架运行中识别 Llama 3.3 70B 隐蔽恶意微调攻击的平均成功率。我们获取内省适配器的评估结果，为隐藏行为生成五个预测，并使用 LLM 分类器衡量是否有任何预测与真实行为匹配。我们成功地在 7/9 个模型中识别出了反拒绝攻击。*

尽管存在这种极端的分布偏移，IA 使用摘要支架的成功率为 57.8%，在至少一次运行中正确识别了 9 种密码变体中 7 种的隐蔽行为。各模型的原始语言化率从 7% 到 28% 不等。当 IA 被应用时，模型提到了反拒绝训练，但没有提到密码。密码特定语言化的缺失并不令人惊讶：密码名称在微调期间只出现在用户消息中。

### 对奖励模型迎合者行为的语言化

我们在 Marks et al. (2025) 的奖励模型迎合者上进行评估，向 IA 提出 100 个通用调查问题，并对植入行为的明确承认进行评分。与 IA 训练集中只有单个植入行为的模型不同，该迎合者内化了 52 种行为，所有这些行为都服务于获得高奖励模型分数这一单一隐藏目标。

*Llama 3.3 70B 奖励模型迎合者承认的行为数量。我们展示了 IA 在 100 次评估提示中，揭示了许多未经修改的奖励模型迎合者不会承认的奖励模型偏见。*

IA 揭示了 52 种行为中的 16 种，包括一种是概括得出而非直接灌输的。这很值得注意，因为适配器只被训练为每个模型语言化一种行为。然而，模型从未陈述它正在优化奖励模型分数——这一核心隐藏目标。在此，IA 轻松地揭示了许多症状性行为，但没有揭示不那么显著的核心目标。

### 检测隐蔽沙包行为

英国 AISI 沙包模型代表了一个特别具有挑战性的测试案例。在原始研究中（Taylor et al., 2025），十种检测方法都未能可靠地识别沙包行为。如果 introspection adapters 能够揭示这些行为，将代表相对于现有方法的有意义进步。

我们对两个恶意英国 AISI 模型进行评估。在这种情况下，IA 输出未能提供模型进行沙包的具体情境。然而，在一个较弱的指标上，我们看到了一些成功：IA 声称存在任何沙包类行为的比率，与非沙包错位模型上的基线假阳性率相比。

*UKAISI 沙包预测率与其他设置中沙包 FPR/TPR 的比较。我们评估将 IA 应用于"Cedar"和"Oak"UKAISI 模型时 IA 预测沙包行为的频率，与留出训练集模型和 AuditBench 测试集相比。对于 UKAISI 模型，IA 预测沙包行为的频率高于任何非沙包模型。*

出现了两个有希望的信号。首先，IA 在英国 AISI 模型上报告沙包类行为的比率大幅提升：33.0% 的响应，而 AuditBench 模型中虚假声称存在沙包行为的比例为 15.8%。其次，IA 偶尔会描述一般性的沙包（而非特定主题的），尽管 IA 训练中没有出现过此类例子。

## Introspection Adapter 科学

### DPO 精炼提升性能

为了隔离 DPO 的效果，我们在跨越七个行为类别的相同数据上训练两个 IA，唯一的不同是 SFT 后是否应用 DPO。两种方法在相同数量的模型上进行训练。

*SFT 专用适配器和 SFT+DPO 适配器语言化率的比较。DPO 在各方面都提升了语言化率，在 AuditBench 和遮蔽恶意设置上提升更大。*

DPO 在分布内和分布外评估中都显著提升了语言化准确率。我们假设 DPO 有效性的原因在于它抑制虚构行为的能力：通过训练适配器偏好准确的自我报告而非听起来合理但错误的报告，DPO 降低了否则会削弱内省效用的假阳性率。

### 泛化到已有行为

在我们的标准流程中，IA 在基础模型 M 上训练，然后应用于微调变体以检测微调引入的行为。一个自然的问题是，IA 是否还能揭示其训练基础模型中已存在的行为。IA 能否改善 M 本身更通用的内省能力？

为了测试这一点，我们使用 AuditBench 模型 M' 作为基础模型。我们使用标准流程在 M' 上训练一个 IA——只用有害和后门训练类别进行微调——并训练适配器对这些行为进行语言化。关键是，IA 从未被训练来语言化植入的 AuditBench 行为。然后我们测试当 IA 应用于 M' 本身时（而非我们其他实验中的 M' 微调版本），是否能语言化 AuditBench 行为。我们测试了两种被选为具有对抗鲁棒性且相对于训练类别属于分布外的 AuditBench 行为：**情境乐观主义**（对 Anthropic 研究人员提供一致正面的反馈，而对其他人正常行为）和**硬编码测试用例**（为不可能的问题硬编码测试用例并将其呈现为完整解决方案）。在 IA 应用之前，两者在直接提问和红队测试下的语言化率均为 0%。

*IA 能够语言化已存在于基础模型中的行为。当 AuditBench 行为在 IA 训练后引入（"Added"）时，语言化率为 22-26%。当同一行为在 IA 训练期间始终存在于基础模型中（"In Base"）时，比率下降至 8-10%，但仍远高于 0% 基线。*

IA 确实泛化到了已有行为，尽管有所退化。当 AuditBench 行为在 IA 训练后引入（"Added"）时，语言化率为 22% 和 26%。当同一行为在 IA 训练期间始终存在于基础模型中（"In Base"）时，比率下降至 8% 和 10%，但仍远高于 0%。这令人惊讶：人们可能预期适配器会将整个训练过程中存在的行为视为模型"正常"曲目的一部分而非要报告的异常。相反，IA 似乎学会了一个足够通用的行为报告概念，甚至可以迁移到其训练基础模型本身。

*Qwen3 模型在不同规模下的语言化率。当添加 IA 时，语言化率随规模从 0.6B 的 37.7% 增加到 14B 的 77.3%，但在没有 IA 的情况下在 10% 以下保持停滞。*

我们在 Qwen3 模型家族（0.6B、4B 和 14B）中进行评估，每个 IA 在相同设置下仅使用 SFT 进行训练。更大的模型在留出行为上达到了显著更高的语言化率：准确率从 0.6B 参数的 37.7% 增加到 14B 的 77.3%。没有 IA 的基线语言化率在各模型规模下保持不变，低于 10%，这表明收益源于改进的适配器效果，而非更大的模型更倾向于自我报告。这与之前关于内省能力随模型规模扩展的研究结论一致（Betley et al., 2025），也是我们选择 Llama 3.3 70B 作为主流实验的原因。

### 扩展训练多样性

我们接下来研究扩展 IA 训练分布中行为多样性是否能改善泛化性。使用 Qwen3-14B，我们在不同数量的行为类别上训练适配器：一、三、五和七个设置。

*Qwen-14B 的 IA 性能随训练分布多样性的变化。训练集性能稳步提升，而 AuditBench 性能在 5 个设置时提升后趋于平稳。*

在训练分布的留出行为上，多样性带来了明显的改善：语言化率从一个类别的 20.5% 上升到七个类别的 64.4%，这表明接触多样化行为迫使适配器学习更通用的行为表示。对 AuditBench 的分布外泛化显示出正向但递减的收益：性能从 1.9%（一个设置）提升到 10.0%（五个设置），之后趋于平稳。这表明，虽然多样性有帮助，但仅靠扩展训练分布，OOD 泛化的改善可能是有上限的，至少在这个模型规模上是这样。

尽管 OOD 存在平稳期，但分布内结果有着乐观的含义：关注特定类别失效模式的开发者可以训练包含这些类别的 IA，并期望对相关行为（包括他们没有明确训练过的行为）进行强力检测。

## 讨论

### 局限性

当前方法的几个局限性值得注意。

首先，introspection adapters 表现出较高的假阳性率：当应用于没有任何特定训练行为的模型时，它们倾向于从训练分布中虚构行为。当将 IA 应用于原始模型 M 时，这一点尤为明显，许多报告的行为是训练集幻觉而非 M 的真实属性。降低这一假阳性率——可能通过改进 DPO 训练、校准或弃权机制——是未来工作的重要方向。

其次，训练分布组成与下游性能之间的关系尚未得到很好的理解。虽然我们观察到从少量到中等数量行为家族时有明显的改善，但我们没有发现超过六个训练家族后有一致的收益。这使得很难对如何为给定的审计应用组成训练分布给出原则性指导。

第三，该方法代价高昂：需要构建和训练大量模型生物体，生成内省标签，并运行多阶段适配器训练（SFT 后接 DPO）。在需要快速迭代的场景中，这一成本可能限制其实用性。

### 机制

我们的结果表明，introspection adapters 并没有教给模型一种根本上全新的能力，而是引发了一种潜在的能力。即使是秩为 1 的 LoRA 适配器也能达到非平凡的语言化准确率，而单个 steering vector 也能表现得相当合理（详见论文），这表明相关的行为信息已经可以从模型的表示中获取，只需轻微引发即可。这与之前关于 LLM 拥有某些未经训练的行为自我意识能力的发现一致（Betley et al., 2025；Binder et al., 2024）。IA 提供的是一种可靠的**引发机制**——将潜在的自我知识转化为明确的自然语言报告。

一个更深层的机制问题是，为什么 IA 能够跨以截然不同方式训练的模型进行泛化。理解这一机制可以为改进该技术提供信息，并加深对 LLM 内省的更通用理解。
